Preprocessing

few additional refinements to address common preprocessing challenges in text data:

  1. Synonym Mapping: Replace synonyms (e.g., "good," "great," "excellent") with a unified term to reduce semantic redundancy.
  2. Named Entity Standardization: For words with multiple common forms, standardize terms to prevent fragmentation (e.g., "movie" and "film" could both become "movie").
  3. Handle Negations: Combine terms following negation words (e.g., "not good" "not_good") to retain sentiment nuances.
In[9]:
# Synonym mapping for common terms to improve clustering
synonym_dict = {
    "good": "positive",
    "great": "positive",
    "excellent": "positive",
    "bad": "negative",
    "terrible": "negative",
    "movie": "film",
    "film": "film",
    "picture": "film"
}

def preprocess_text(text):
    # Convert text to lowercase and tokenize
    tokens = word_tokenize(text.lower())
    
    # Remove stopwords and non-alphabetical tokens
    stop_words = set(stopwords.words('english'))
    tokens = [word for word in tokens if word.isalpha() and word not in stop_words]
    
    # Lemmatize tokens
    lemmatizer = WordNetLemmatizer()
    tokens = [lemmatizer.lemmatize(word) for word in tokens]
    
    # Apply synonym mapping
    tokens = [synonym_dict.get(word, word) for word in tokens]
    
    # Handle negations by joining "not" with the following word, if present
    tokens = ['_'.join([tokens[i], tokens[i+1]]) if tokens[i] == 'not' and i+1 < len(tokens) else tokens[i] 
              for i in range(len(tokens) - 1) if tokens[i] != 'not'] + ([tokens[-1]] if tokens[-1] != 'not' else [])
    
    return ' '.join(tokens)

def find_optimal_clusters(tfidf_matrix):
    inertia = []
    for k in range(1, 10):
        kmeans = KMeans(n_clusters=k, random_state=42)
        kmeans.fit(tfidf_matrix)
        inertia.append(kmeans.inertia_)

    plt.plot(range(1, 10), inertia, marker='o')
    plt.xlabel('Number of Clusters (k)')
    plt.ylabel('Inertia (WCSS)')
    plt.title('Elbow Plot for Optimal k')
    plt.show()

Given that the clustering is showing a significant imbalance (9 reviews in one cluster and only 1 in the other), there are a few areas to investigate and refine to improve the clustering so it aligns more closely with the sentiment labels. Heres an outline of the issues and refinement suggestions:

Observations and Issues Imbalanced Clusters: With 9 reviews clustered together and only 1 review in a separate cluster, K-means is likely treating most reviews as similar, failing to differentiate between positive and negative sentiments. NMF Features: NMF (Non-negative Matrix Factorization) can sometimes struggle with fine-grained sentiment distinctions if the input representation does not effectively capture the nuance in positive vs. negative sentiments.

Steps for Refinement

  1. Increase n_components in NMF: Instead of n_components=4, we can try a higher number, such as n_components=6 or n_components=8, to capture more distinct topics and potentially reveal positive vs. negative sentiment separation.
  2. Refine Clustering with Sentiment Labels in Mind: The imbalance might be mitigated by adding a sentiment-based pre-clustering step or tuning parameters to enhance feature separability.
  3. Alternate Clustering with DBSCAN: Since K-means struggled to create balanced clusters, DBSCAN (Density-Based Spatial Clustering of Applications with Noise) might work better as it can help identify core and outlier points, potentially splitting positive and negative sentiments more effectively.

Analyzing Refinements

With the increased NMF components from 4 to 6, we hoped to capture more nuanced subtopics that might help differentiate positive and negative sentiments within the reviews. The K-means clustering yielded slightly more balanced clusters (0: 7 and 1: 3), but the Silhouette Score (0.118) and the Adjusted Rand Index (ARI) score (0.071) remain low, indicating that the clustering still does not align well with the true sentiment classes.

Using DBSCAN with eps=0.5 and min_samples=2, we found that it failed to group data effectively, resulting in multiple small clusters (3 distinct clusters) and outliers (-1 label), leading to a negative ARI score (-0.111). DBSCAN typically requires more data points or well-separated clusters, so it is often less effective for short text data like this.

Cluster Misallignment

The three mismatched reviews from K-means clustering (Reviews 2, 3, 8) in cluster 0 were labeled as positive but placed within the same cluster as negatives. Here are some key observations from the top words in each topic for these cases:

Topic Words Overlap Across Sentiment Contexts: Words like dark, film, story, and scene appear in both positive and negative contexts, making it hard for the clustering algorithm to distinguish sentiment without further context. Bi-gram Limitations: Although bigrams helped, phrases like positive batman or negative people may still lack the expressive strength to represent sentiment distinctly in the feature space. This is evident as some topics dont clearly separate positive and negative opinions.

Vocabulary Issues and Polysemy

The terms dark, time, scene, etc., do not consistently signal either positive or negative sentiment due to polysemy (multiple meanings) and context-dependency. For instance:

Dark can imply a gritty atmosphere (often positive in film reviews) or imply negativity if associated with depressing. Positive words in some contexts can be neutral in others, e.g., positive batman or good cinematography are less clear indicators.

Initial Issues with Simple Clustering

The clustering process did not yield the desired results for several reasons, which fall into two main categories:

  1. Term Variants and Entity Issues

    • Inconsistent Treatment of Similar Terms: Terms that should be semantically identical, like "batmen" and "Batman," or even broader sentiment-related variations, such as "loved" and "love," were treated as distinct by the algorithm. These variations can lead to fragmented clusters, especially in sentiment analysis, where consistent terminology is crucial for distinguishing positive and negative language.

    • Missed Entity Connections: Terms like Bruce Wayne, Batman, Gotham, and Riddler hold specific connotations but are sometimes separated in the document-term matrix, resulting in weaker feature connections. These separations affect the models ability to identify meaningful themes within reviews, contributing to mixed or incoherent clusters.

  2. Concept Representation Limitations

    • Insufficient Conceptual Coverage: Words or phrases that are central to understanding sentiment were not adequately captured in the initial TF-IDF matrix. Phrases like "not recommend" or "strongly disagree" are key indicators of negative sentiment but are treated as separate words, reducing their significance in clustering.
    • Underrepresentation of Contextual Nuance: The TF-IDF model and basic NMF clustering do not capture context or semantic relationships between words. For instance, terms like "dark" and "noir" could indicate a positive review for certain viewers (who enjoy those styles) but could signify negativity for others, leading to sentiment ambiguity in clusters.

Refinements and Observations

To address these issues, we implemented several refinements to improve the clustering results:

  1. Increased NMF Components to Capture Nuance

    • Adjustment: We increased the NMF components from 4 to 6 to capture a broader range of topics, which we hoped would help separate clusters based on sentiment.
    • Outcome: The adjustment revealed distinct topics but did not achieve clear alignment with sentiment, as shown by the Adjusted Rand Index (ARI), which remained low. This result highlights that while additional topics may capture more thematic nuances, they do not necessarily improve sentiment separation due to overlapping language and concepts.

  2. Incorporating Bigrams with TF-IDF

    • Adjustment: To address the issue of phrase continuity (e.g., not recommend), we used bigrams in the TF-IDF matrix, allowing the model to capture more meaningful sentiment expressions.
    • Outcome: The bigram-enhanced TF-IDF provided some improvement in feature representation, but the clusters still showed mixed sentiment alignment. This approach helped identify recurring phrases, yet without context, these phrases were insufficient for reliable sentiment-based clustering.

  3. Doc2Vec with Refined Parameters

    • Adjustment: We introduced Doc2Vec with refined parameters (increased vector_size and window) to capture more semantic relationships and account for word contexts, aiming for a more sentiment-oriented clustering.
    • Outcome: Doc2Vec improved the internal coherence of clusters (as shown by the higher Silhouette Score), indicating that documents within clusters were more thematically similar. However, sentiment alignment remained weak, with the ARI score staying low, suggesting that Doc2Vec effectively grouped documents by general tone and topic but failed to consistently map clusters to sentiment.

  4. Experimenting with DBSCAN

    • Adjustment: We applied the DBSCAN algorithm, which is density-based and could potentially identify outliers or smaller clusters. DBSCANs flexibility in defining clusters by density made it a useful test for isolating unique reviews.
    • Outcome: DBSCAN did identify outliers but produced highly fragmented clusters without strong sentiment alignment, as indicated by a negative ARI score. The result underscores that density-based clustering is insufficient for sentiment clustering when reviews contain overlapping vocabulary but differ primarily in sentiment.
